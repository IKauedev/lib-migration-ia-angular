# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project Overview

**ng-migrate-angularjs-ai** — CLI for migrating AngularJS 1.x → Angular 21 using multi-provider AI (Anthropic Claude, OpenAI GPT, Azure OpenAI, Google Gemini).

**Version:** 2.0.0 | **License:** MIT | **Node:** 18+

## Quick Commands

```bash
npm install              # Install dependencies
npm start                # Run CLI
npm run migrate          # Run migrate command
npm run repl             # Start interactive REPL

ng-migrate config        # Configure AI provider
ng-migrate scan          # Analyze AngularJS project (generates .ng-migrate-*.json files)
ng-migrate migrate-project  # Full project migration
ng-migrate migrate <file>   # Single file migration
```

## Architecture

### Entry Point
- `src/index.js` — Commander-based CLI with all commands registered

### Commands (`src/commands/`)
| File | Purpose |
|------|---------|
| `config.js` | Interactive AI provider configuration (saves to `~/.ng-migrate/config.json`) |
| `env.js` | Environment variable manager (`~/.ng-migrate/.env`) |
| `scan.js` | Static analysis of AngularJS projects, generates 3 JSON files |
| `migrate-project.js` | Full project migration with phase-sorted order |
| `migrate-repo.js` | Clone + migrate remote GitHub/GitLab repos |
| `migrate.js` | Single file migration |
| `analyze.js` | AI-powered qualitative analysis |
| `repl.js` | Interactive code conversion loop |
| `checklist.js` | Generate migration checklist |

### Utilities (`src/utils/`)
| File | Purpose |
|------|---------|
| `ai-providers.js` | Multi-provider client builders (Anthropic, OpenAI, Azure, Gemini, OpenAI-compatible) |
| `ai.js` | Core AI migration functions (`migrateWithAI`, `analyzeWithAI`) |
| `config-manager.js` | Load/save config from `~/.ng-migrate/config.json` |
| `env-loader.js` | Load `.env` file into `process.env` |
| `project-scanner.js` | Static analysis: detects AngularJS patterns, builds dependency graph |
| `ng-checker.js` | Angular CLI installer, git clone, `ng new` scaffolding |
| `deps-migrator.js` | Migrates package.json dependencies |
| `parser.js` | Parse AI responses (extract code blocks, detect types) |
| `report.js` | Generate migration reports |
| `git-providers.js` | GitHub/GitLab API wrappers for repo cloning |
| `langchain-provider.js` | LangChain integration layer |
| `debug.js` | Debug mode logging (enabled via `--debug` flag) |

## Configuration Files

### User-level (`~/.ng-migrate/`)
- `config.json` — AI provider config (provider, model, API keys)
- `.env` — Environment variables (API keys, tokens)

### Project-level (generated by `scan`)
- `.ng-migrate-analysis.json` — Project summary, file list, complexity, migration phases
- `.ng-migrate-registry.json` — Symbol map (AngularJS name → Angular 21 class name), routes, modules
- `.ng-migrate-deps-graph.json` — Dependency graph between files

## Migration Phases (automatic ordering)

1. **Services & Factories** — No dependencies in most cases
2. **Filters → Pipes** — Depend only on services
3. **Directives & Components** — Depend on services and pipes
4. **Controllers** — Depend on all above
5. **Templates HTML** — Depend on components
6. **Routing & Modules** — References all previous phases

## AI Providers

Configured via `ng-migrate config`:
- **Anthropic** (`claude-opus-4-5`) — Recommended for best migration quality
- **OpenAI** (`gpt-4o`) — GPT-4o and variants
- **Azure OpenAI** — Requires endpoint, deployment, API version
- **Google Gemini** (`gemini-2.0-flash`) — Gemini 1.5 Pro/Flash
- **OpenAI-compatible** — Ollama, Together AI, Groq, etc.

## Key Patterns

### AngularJS → Angular 21 Mappings
| AngularJS | Angular 21 |
|-----------|------------|
| `.controller()` | `@Component` standalone with Signals |
| `.service()` / `.factory()` | `@Injectable({ providedIn: 'root' })` |
| `.filter()` | `@Pipe({ standalone: true })` |
| `$scope` | `signal()` / `computed()` / `effect()` |
| `$http` | `HttpClient` with `inject()` |
| `$routeProvider` | `provideRouter()` |
| `ng-repeat` | `@for` |
| `ng-if` | `@if` |
| `ng-model` | `[(ngModel)]` or signal binding |

### Debug Mode
Run with `--debug` flag to enable detailed logging:
```bash
ng-migrate scan --debug
```

Debug logs show: files read/created, AI requests/responses, phase progression.

## Output Structure

### Default mode (separate folder)
```
<project>-angular21/
├── src/app/           # Migrated files
├── angular.json       # Generated scaffold
├── tsconfig.json      # Generated config
├── package.json       # Updated dependencies
└── MIGRATION_REPORT_*.md
```

### In-place mode (`--in-place`)
```
<project>/
├── src-angularjs-backup/  # Original files backup
├── src-angular21/         # Migrated files
├── angular.json
├── tsconfig.json
└── package.json
```

## Testing Notes

- Progress bars via `cli-progress` (disabled in debug mode)
- File operations use native `fs` module
- Git operations via `simple-git`
- AI calls use provider SDKs directly (not LangChain by default)
